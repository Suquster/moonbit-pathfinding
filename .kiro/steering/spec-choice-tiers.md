# 选择呈现规则：丰富度 / 难度 / 广度 三档递进

只要向用户呈现任何"怎么做 / 做到什么程度"的选择（尤其是 spec 方向深化、功能范围、实现强度），**必须**附带以下三档递进选项，且默认推荐最高档：

- 🟢 档位 1「夯实基础」：补齐核心高频能力。中等广度、低难度，快速可用。
- 🔵 档位 2「进阶完善」：功能完整 + 算法深度 + 基准 + 可解释文档。较高广度与难度。
- 🟣 档位 3「业界顶尖（旗舰）」【默认推荐】：最大广度与难度，前沿/高级特性、性能优化、paper-to-code 可追溯、开源对标、完整 PBT、三后端一致性。

递进含义：档位越高 → 丰富度↑、难度↑、广度↑。允许用户对不同方向选不同档位；也允许"全部统一用某档位"。全程中文沟通，spec 文档亦用中文。
