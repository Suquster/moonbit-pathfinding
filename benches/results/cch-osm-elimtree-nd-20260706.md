# CCH 消除树查询 + 嵌套剖分（ND）序真实 OSM 路网证据（2026-07-06）

采集命令（预热 3、采样 12、中位数、全查询代价对拍护栏）：

```bash
moon bench --target native -p benches/advanced_bench -f osm_cch_bench.mbt
```

新增实现：`src/directed/cch.mbt` 的 `query_dist_tree`——消除树无堆查询
（Bauer et al. / CCH 论文）：沿消除树父链（`ep[v]` = 最低 rank 上邻）从
s、t 各自走到根，按 rank 升序松弛全部上邻，无优先队列、无 stall。语义由
`src/directed/cch_test.mbt` 100 迭代差分 PBT（含 ND 序变体与换权
recustomize）与本基准 48 组全量对拍守卫。

## 结论（诚实口径）

- **消除树无堆查询有效且更快**：厦门 114 µs（比堆版 CCH 149 µs 快
  1.30×），北京 728 µs（快 1.07×）。48 组×2 城全量对拍一致。相对
  CH（38/132 µs）仍慢——差距根源是骨架密度而非查询机制。
- **ND 序当前实现（BFS 层分隔符）质量不合格，属已知问题**：厦门骨架边
  4,022,969 vs min-degree 197,723（20× 填充），构建 120.6 s、
  customization 39 s、查询反而更慢；北京规模下构建不可行（>90 分钟未
  完成，已从采集中剔除）。根因：BFS 层作分隔符在路网上过宽（数百~数千
  节点成团），弦图补全 O(sep²) 爆炸。修复方向：真正的小分隔符
  （inertial flow / flow-based refinement）+ 构建端用桶合并替代
  O(deg) 数组插入。**生产推荐 min-degree 序**，ND 序保留为实验路径
  （小图正确性由 PBT 持续守卫）。
- CCH 主卖点不变：换权成本北京 1.89 s vs CH 全量重建 25.1 s
  （13.3×），厦门 111 ms vs 2.13 s（19.2×）。
