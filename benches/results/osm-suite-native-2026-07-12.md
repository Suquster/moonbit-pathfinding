# 真实 OSM 路网加速结构全家福 — native 基准（2026-07-12 复测）

> 复测环境：Linux x86_64（Devin 会话 VM，2 vCPU），moon 0.1.20260703 native。
> 命令：`moon bench -p benches/advanced_bench --target native`。
> 数据集：OpenStreetMap 厦门驾车网（23 925 节点 / 54 151 有向边）、
> 北京驾车网（163 501 节点 / 406 591 有向边），osmnx 提取，权重=道路米长。
> 统计口径：48 查询对，预热 3，采样 12，取中位；全部与 Dijkstra 逐位代价对拍一致。

## CH vs Dijkstra（osm_ch_bench）

| 路网 | Dijkstra 每查询 | CH 每查询 | 中位加速比 | CH 预处理 |
|---|---|---|---|---|
| 厦门 | 1286.3 µs | 26.86 µs | **47.9×** | 762.7 ms |
| 北京 | 11627.7 µs | 116.66 µs | **99.7×** | 7141.4 ms |

## ALT / CH / Hub Labeling / PHAST / many-to-many（osm_alt_bench）

| 指标 | 厦门（k=8） | 北京（k=16） |
|---|---|---|
| 双向 Dijkstra 每查询 | 715.0 µs | 7178.0 µs |
| ALT 双向 A* 每查询 | 748.9 µs（0.95×） | 1113.4 µs（**6.4×**） |
| CH 每查询 | 39.0 µs（18.3×） | 144.9 µs（**49.5×**） |
| Hub Labeling 每查询 | 0.297 µs（**2409×**） | 0.639 µs（**11242×**） |
| PHAST 一到全每源 | 332.3 µs（vs 全量 Dijkstra 6.07×） | 2513.5 µs（6.33×） |
| many-to-many 64×64 bucket | 5.50 ms（vs 逐对 CH **29.3×**） | 34.24 ms（**16.0×**） |
| RPHAST 每源 | 41.0 µs（vs PHAST 8.10×） | 251.6 µs（9.99×） |

全部代价对拍一致（单向/双向 Dijkstra、ALT、CH、HL、PHAST 树、many-to-many、RPHAST）。

## CCH 换权成本优势（osm_cch_bench）

| 指标 | 厦门 | 北京 |
|---|---|---|
| CH 全量重建 | 2287.9 ms | 27338.7 ms |
| CCH customization（换权）中位 | 121.0 ms（**18.9× 快于重建**） | 2074.5 ms（**13.2×**） |
| CCH 每查询 | 169.0 µs | 858.4 µs |
| 消除树无堆查询 | 111.9 µs（min-degree 序） | 809.2 µs |

原权重与换权后代价对拍均一致。

## 大规模合成网格 CH（ch_csr_bench）

- 250 000 节点 / 998 000 有向边 4-连通加权网格；
- Dijkstra 每查询 20733.6 µs → CH 112.6 µs，中位加速比 **184.1×**（门槛 ≥100×：PASS）；
- 24 查询对代价对拍全一致。

## 解读

- HL 查询把北京全市任意点对最短路代价压到亚微秒（0.64 µs），换来的是
  预处理 29.5 s 与平均标签长 128.9 —— 时间/空间换查询的完整谱系
  （Dijkstra→ALT→CH→HL）在同一真实路网上可复现对比。
- CCH 在"权重频繁变化"场景（实时路况）用 13–19× 于 CH 重建的
  customization 优势换取 4–6× 查询慢因子，为经典 metric-independent
  预处理权衡提供了实测数据。
- 与 2026-07-05 基线相比各项加速比同量级（CH 北京 99.7× vs 104×，
  合成 250k 184× vs 161×），确认结果在不同机器间可复现、无回归。
